The shift

AI agents are becoming the primary interface to data.

Analysts and domain experts increasingly interact with data through AI agents rather than dashboards and SDKs. The connection layer between agents and external data is consolidating around one open standard.

MCP
Model Context Protocol

The emerging open standard for connecting AI agents — Claude, ChatGPT, and enterprise copilots — to external data sources and tools. Now backed by every major LLM provider.

What we propose to build

Two pillars that make EarthDaily agent-native.

Together they cover both directions of the data flow — EarthDaily's APIs out to agents, and clients' internal data in.

Pillar 01

Earth Daily MCP Server

A production-grade MCP wrapper around EarthDaily's APIs — so any AI agent can query satellite imagery, run analytics, and retrieve insights through natural language.

  • Careful design of LLM-handled vs. compute-offloaded operations
  • Token-efficient return of GeoTIFFs, STAC items, and raster data
  • Compatible with Claude, ChatGPT, and enterprise copilots
Pillar 02

Client Data Connector Framework

An interface that lets EarthDaily's customers easily expose their own internal data — field boundaries, yield records, claims data, asset registries — as MCP connectors.

  • Vertical-specific templates (ag, insurance, mining, forestry)
  • Schema-agnostic semantic mapping layer
  • Agent reasons across EarthDaily + client data in one workflow
